### 1. PROJECT OVERVIEW
- Language, framework, dependencies: Python 3.10+, uv/pip package manager. Core dependencies include `anthropic`, `openai`, `pydantic`, `mcp` (Model Context Protocol), `prompt-toolkit` (for CLI), and `tiktoken`. Tests use `pytest` and `pytest-asyncio`.
- Entry point and how the app starts: The primary entry point is `mini_agent/cli.py:main()`, which initializes configuration, loads base tools (Bash, File tools, MCP, Skills), sets up the LLM Client, and launches the interactive `PromptSession` loop calling `agent.run()`.
- Current architecture: A modular, event-driven agentic framework built for MiniMax models. It uses a decoupled structure: `LLMClient` handles API communication with specific parsing for `thinking` blocks, `Agent` orchestrates the ReAct loop (managing state, limits, and background telemetry), and a dynamic tools system loads built-in features, dynamic Claude skills, and MCP servers. A 2-stage prompt refinement pipeline intercepts user inputs before hitting the main agent loop.

  - What's wrong: PermissionError on Windows `Path(f.name).unlink()` because Windows holds the temp file handle during MCP async teardown.

  - Severity: major (can cause M2.7 context window crashes).
